# 🧠 2. 핵심 로직 복기: '조건부 챔피언 갱신' 과 '절댓값(abs)'
단순히 '제일 큰 값' 이 아니라 '특정값(100 에 가장 가까운 값)' 을 찾을 때는 abs() 를 써서 '거리(차이)' 를 구한뒤에 abs() 덮어서 실제로 딱 정수값만
비교하는 것이 핵심이였다. <br> <br>

```Python
# 백준 2851번 핵심 로직 : 100 에 더 가까운 수 찾기
diff_current = abs(100 - total_num)
diff_prev = abs(100 - prev_num)

# 거리가 같거나, 현재 숫자가 100 에 더 가깝다면 갱신
# (<= 를 사용해서 거리가 같을 땐 더 큰 수가 정답이 되도록 처리)
if diff_current <= diff_prev:
    print(total_num)
else:
    print(prev_num)
```


# 🚨 3. 피드백 및 부족했던 점
- 리스트 입력받기 문법의 혼동 (map vs list(map))
    - a, b = map(int, input().split()) 
    이런 케이스는 보통 변수 개수가 정해져 있어서 쪼개서 넣을 때 사용하고 <br>
    특징은 map 은 그 자체로는 리스트가 아닌 'map 객체' 이기 때문에 인덱싱이 불가능함 <br>

    - my_list = list(map(int, input().split()))
    이런 케이스는 입력받은 값들을 통째로 하나의 리스트 ([1,2,3]) 로 묶고 싶을 때 사용하고 <br>
    특징은 바깥에 반드시 list 를 씌워줘야 우리가 아는 정상적 리스트 형태가 됨 <br>

- 약수와 배수의 위치 혼동 : A % b == 0 식에서 앞뒤 숫자의 역할을 헷갈렸음. 내가 '배수' 를 찾고싶다면 찾는 수 % 리스트숫자 == 0 이 되어야 함.

- 무한 탐색 루프의 부재 : 정답이 언제 나올지 모를 때는 while True: 로 감싸서 for 문을 가두고 1씩 무한정 키워나가는 방식을 채택해야 함.
