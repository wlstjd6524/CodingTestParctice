# 🧠 2. 핵심 로직 복기: '역발상(소거법)'과 'set 자료형의 위력'
숫자의 자릿수를 다루거나 알파벳의 빈도수를 셀 때, 복잡한 수학 공식보다 데이터 형태를 바꿔서 (형변환) 접근하는 것이 파이썬 다운 해결책임을 느낌.

```Python
# 4673번: "내가 생성자인가?" (X) -> "내가 만든 숫자를 명단에서 지우자!" (O)
generated_numbers = set() # 탈락자 명단 (중복 자동 제거, 검색 속도 최상)
for i in range(1, 10001):
    num_sum = i + sum(int(char) for char in str(i)) # 분해합 계산
    generated_numbers.add(num_sum) # 탈락!


# 1157번: 챔피언(최빈값) 찾기 핵심 로직
word = input().upper()       # 입력 즉시 원본을 대문자로 덮어쓰기! (word.upper()만 쓰면 안 됨)
unique_letters = list(set(word)) # set으로 중복 알파벳 제거 후 후보군 생성
```



# 🚨 3. 피드백 및 부족했던 점
- 문자열 불변성의 망각 : word.upper() 를 사용할 때 문자열에 덮어씌우지 않아서 대문자 반영이 안됐음.
    - word = word.upper() 식으로 변수 하나에 지정해서 저장해줘야됨.

- 중복을 제거해야 하는 케이스에도 하나씩 문자를 다 세려고 함 : set() 미활용
    - set() 이 중복을 제거한다는 사실을 인지하고 다음부터는 중복이 큰 의미가 없는 경우는 set() 으로 묶어 중복을 제거
