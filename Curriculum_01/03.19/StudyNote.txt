2. 없으면 -1 을 반환하는 .find()
    - .index() 함수와 비슷하지만 결정적인 차이가 존재했던 find()
    .index() 는 찾는 문자가 없으면 프로그램이 ValueError 를 뱉지만 find() 의 경우는 -1 을 반환함.

    - 문자열 안에서 특정 문자의 위치를 찾아야 하는데, 그 문자가 없을 수도 있으니까 (만약 없는데 .index() 를 사용해버리면 프로그램이 종료됨)
    이럴 때 사용하면 좋고

    - .find() 는 리스트에서는 사용이 안됨, 오직 문자열에서만

    ex)
    text = "hello"
    print(text.find('e'))       # 1 출력
    print(text.find('z'))       # -1 출력, z라는 문자가 없음

3. 아스키코드 형태 이용해보기

    ex)

    # 문자를 숫자형태는
    print(ord('a'))             # 97 출력

1. 지수 표현 방식
    - 파이썬에서 e나 E를 이욯한 지수 표현 방식을 이용할 수 있음 ex) 1e9 -> 10의 9제곱의 의미
    - 공식은 '유효숫자e지수 = 유효숫자*10지수' 형태

2. 실수형 더 알아보기
    - 우리가 사용하는 10진수 체계에서는 0.3 + 0.6 을 더한 값이 0.9 로 바로 떨어질 수 있음
    - 하지만 컴퓨터는 2진수를 사용하기 때문에 이런 실수형을 더 했을 때 0.9 를 바로 정의 하기가 불가능함 그래서 0.9와 거의 가까운 값을 내뱉는 구조
    그래서 파이썬에서 다음과 같이 수를 비교하면 False 를 내뱉을 거임.
    
    ex)
    a = 0.3 + 0.6
    b = 0.9

    if a == b:
        print("True")
    else:
        print("False")
    
    - 이런 케이스의 경우 실수를 더했을 때 비교해야될 순간이 올 수도 있는데 이렇게 False 를 뱉어버리면 구조의 한계로 개발의 전진이 불가능 할 수도 있음.
    - 이럴 때 round() 함수를 사용하여 소수점 몇 째 자리에서 짤라버리는 구조로 정확히 수를 맞춰줄 수 있음

    ex)
    123.456 을 소수 셋째 자리에서 반올림 하려면 round(123.456, 2) -> print = 123.46 이 출력됨
    얘를 123.46 으로 선언된 변수랑 비교하면 True 값을 받을 수 있음
