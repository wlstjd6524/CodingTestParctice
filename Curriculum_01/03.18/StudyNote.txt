1. Data Type 의 성질과 코드 다이어트
    - 정수형은 단단한 덩어리 같은 존재라서 슬라이싱이 불가능함 [::-1] 같은거. 그래서 입력받은 문자열 상태에서 뒤집고 바꾸고 요리하고 조리하는 걸
    다 해놓은 상태에서 마지막에 정수형으로 바꾸는게 현명함.

    - 또한 무조건 조건문으로 접근하는게 아니라 max(int(a[::-1]), int(b[::-1])) 식으로 값을 줘버리면 애초에 조건이 걸리기 때문에
    코드 다이어트도 가능

2. 문자열 검사는 자르지말고 '바꿔치기'
    - 크로아티아 알파벳(2941) 을 풀 때 특정 패턴을 찾으면서 인덱스로 하나씩 비교하면서 건너뛰는 노가다 대신에, 패턴 덩어리를 찾아서 "치환" 하는
    패턴도 같이 사용해보려고 노력해야 됨.

    - .replace() 를 사용해서 치환 하는데, 이 함수의 특징은 찾는 문자가 없으면 에러 없이 알아서 무시하니까 불필요한 if 문 검사도 안할 수 있음
    (1번에서 max() 를 사용하면서 조건문을 pass 한 논리와 같은 조건임.)

    - word = word.replace(cro, '*')

3. 상태 추적(플래그)과 초기화의 중요성
    - 다른걸 다 떠나서 직전값을 비교하는데 범위 초과의 에러가 발생할까봐 고민을 많이 했었는데 "list[-1]" 을 사용하면 직전 값을 쉽게 비교할 수 있다.

    - Flag 기법이라는 것도 다시 리마인드 하게 되었는데 is_grup: True 형식으로 지정하고 어떤 조건문을 다 통과했을 때 얘가 아직도 True 면 뭐 숫자를
    올려서 그걸 출력하는 기법도 잘 익히게 되었다.
